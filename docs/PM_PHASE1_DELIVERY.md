# 第一期优化交付清单（PM → ARCH）

**文档角色**：产品经理输出，供架构师拆模块、估时、定技术方案。  
**产品**：青砚（全中文 AI 工作助理 + 工作台）。  
**一期目标**：在**不扩张业务域**的前提下，提升**可理解性、一致性、可信度与核心工作流体验**，为后续行业模块（工艺单、多环境 Prompt/知识库等）打壳。

---

## 1. 需求理解（当前产品是什么）

### 1.1 目标用户与价值主张

- **现阶段**：偏「个人 / 小团队效率」——任务与项目、日历与提醒、AI 辅助拆解工作、收件箱快速丢想法。  
- **代码中已预埋**：**组织**、**项目成员/环境**、**Prompt/知识库发布流**、**百叶窗工艺单**——更接近「出口厂家 / 项目制协作」方向，与 README「后续平台化」一致。  
- **接手风险**：侧栏**全部入口外露**，但各模块**成熟度不均**，新用户会以为一切已就绪。

### 1.2 当前能力矩阵（实现深度粗分）

| 域 | 能力 | 实现深度（PM 判断） |
|----|------|---------------------|
| 认证 | 注册/登录、JWT Cookie、中间件保护 | **高** |
| 工作台 | 大盘统计、任务片段、日历聚合等 | **高**（单文件体量大，维护成本高） |
| 任务 | 列表、筛选、弹窗 CRUD、逾期提示、提醒标记等 | **高** |
| 项目 | 列表、详情、成员、环境、链到 Prompt/知识库 | **中高** |
| 组织 | 列表、创建、详情 | **中** |
| 收件箱 | 本地列表 + 调 AI 流式解析 + 任务建议卡片 | **中高**（与助手能力重叠） |
| AI 助手 | 对话流式、任务建议、无 Key 引导 | **中高**（引导文案偏「改 .env」，与线上用户场景不符） |
| 搜索 | 顶栏搜索任务/项目 | **中** |
| 提醒 | API 与头栏展示依赖 | **待 ARCH 确认与任务/日历闭环** |
| 日历 | 本地事件 + Google OAuth 同步 | **中高**（依赖环境配置与 Google 审核策略） |
| 设置 | Google 连接说明、状态 | **中** |
| 工艺单 | 独立列表/新建/编辑/计算/导出等 | **中**（行业强相关，与「通用工作台」并列） |
| Prompt/知识库 | 多页面与 API | **中**（偏 B 端配置台，学习曲线陡） |

### 1.3 关键矛盾（一期要解决的产品问题）

1. **认知负荷**：导航上「全能型产品」，文档与 README 仍像「MVP 三板斧」。  
2. **体验一致性**：登录页与工作台视觉语言不统一；各页表单/卡片/空状态未体系化。  
3. **线上可信度**：AI 配置说明仅提本机 `.env`，**部署用户（Vercel）路径缺失**。  
4. **错误可行动性**：API 401、Google OAuth、AI 失败等，用户常不知下一步。  
5. **维护性**：工作台等超大组件，后续改 UI 或加字段成本高。

## 2. 第一期范围边界

### 2.1 一期 **要做**（优化 / 重构交付）

- **信息架构与导航策略**（不一定删功能，但要「分层/标注/引导」）。  
- **设计系统底座**：色彩、字号阶梯、间距、圆角、按钮/输入/卡片/表格、**中文排版**。  
- **核心壳层**：根布局、侧栏、顶栏、页面标题区、**加载/空状态/错误** 模式统一。  
- **核心工作流抛光**：登录注册 → 工作台 → 任务 → 项目（列表级 + 详情首屏）。  
- **AI 相关可信与引导**：无 Key、失败、流式中断的文案与布局（含**线上环境变量**说明）。  
- **设置与集成**：Google 连接成功/失败路径可读、与文档一致。  

### 2.2 一期 **明确不做**（避免范围爆炸）

- **不做**全新业务域（例如新 CRM、新权限模型）。  
- **不做** Google 正式版 OAuth 审核代填（仅产品侧提示与链接）。  
- **不承诺**工艺单 / Prompt / 知识库 **功能补全**，一期最多 **IA 收敛或「预览」标注**（由 ARCH 评估改动面）。  
- **不做**大规模后端领域模型重构（除非为 UI 阻塞所必需，单独立项）。

| **P0-TASK** | **任务列表核心体验抛光** | 最高频路径 | 列表加载/空状态/筛选/弹窗表单在视觉与交互上与 DS 一致；关键操作有明确反馈（保存成功/失败） |

## 4. 验收与发布（一期结束 PM 怎么签字）

- **回归路径**：注册/登录 → 工作台加载 → 任务创建/编辑 → 项目进入详情 → AI 助手发一条（有 Key）→ 设置页打开无报错。  
- **视觉**：任意两个核心页（任务+工作台）并排截图，**按钮/输入/卡片圆角与字重**一致。  
- **文案**：AI 与设置相关说明**不出现「仅 localhost」单一路径**。  
- **文档**：`README` 或 `docs` 中导航/模块说明与**实际侧栏**一致（或注明「预览模块」）。
