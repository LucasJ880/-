# Intelligence Report 人工审核 SOP

## 1. 审核目标

确保 AI 生成的投标情报分析报告在交付给业务团队或客户前：
- 关键判断正确（无误读原文、无遗漏致命红线）
- 可执行性达标（行动项具体、有负责人和时限）
- 格式规范（12 章节完整、无聊天式退化）
- 不自相矛盾（recommendation / fitScore / 最终结论一致）

## 2. 审核时机

| 触发条件 | 操作 |
|----------|------|
| AI 生成完成（reportStatus = `ai_generated`） | 审核人收到通知或主动检查 |
| 用户上传了新文件并重新生成 | 重新审核 |
| 报告被标记为 `needs_revision` 后重新生成 | 必须再次审核 |
| 在项目页面看到报告 | 随时可点击"开始审核" |

## 3. 审核检查清单（8 个关键点）

审核人应逐项检查以下内容：

### 3.1 原文准确性
- [ ] AI 是否正确理解了招标文件的核心要求
- [ ] 是否有明显误读或曲解原文的情况
- [ ] 引用的原始条款是否准确

### 3.2 致命红线识别
- [ ] 所有合规致命风险是否被识别（如 Buy America、认证资质、保险要求）
- [ ] 严重程度标注是否合理（🔴高 / 🟡中 / 🟢低）
- [ ] 是否有应标注为致命风险但被忽略的条款

### 3.3 GO / NO-GO 判断合理性
- [ ] 决策矩阵的评分是否与分析内容一致
- [ ] 加权总分是否支撑最终结论
- [ ] recommendation（pursue/review_carefully/low_probability/skip）是否合理

### 3.4 行动项具体性
- [ ] 行动清单是否有具体动作（不是"进一步评估"这种空话）
- [ ] 是否有建议负责人、时限和优先级
- [ ] 是否遗漏了关键的准备事项

### 3.5 事实/推断/待确认边界
- [ ] 重要判断是否标注了"已知事实 / 推断 / 待确认"
- [ ] 待确认问题清单是否完整
- [ ] 推断是否有合理依据

### 3.6 供应链现实性
- [ ] 中国采购可行性分析是否贴合实际
- [ ] 海运时间、关税、清关风险是否考虑到位
- [ ] 供应链模式建议是否可执行

### 3.7 结构完整性
- [ ] 12 个章节是否全部存在且有实质内容
- [ ] 技术评估、决策矩阵、行动清单是否使用表格
- [ ] 是否存在聊天式语气或空话套话

### 3.8 内部一致性
- [ ] fitScore 与报告分析内容是否一致
- [ ] summary 与最终结论是否一致
- [ ] 不同章节之间是否自相矛盾

## 4. 审核结论分类

| 结论 | reportStatus | 说明 |
|------|-------------|------|
| 可直接交付 | `approved` → `delivered` | 报告质量达标，可发给业务团队 |
| 小修后交付 | `approved` | 通过审核但附带修改备注，由审核人手动修正后交付 |
| 需要重生成 | `needs_revision` | 存在重大问题，需重新上传文件或等待 prompt 优化后重新生成 |
| 不建议交付 | `needs_revision` | 资料严重不足或项目不适合，报告无法达到交付标准 |

## 5. 审核操作流程

```
AI 生成完成 (ai_generated)
       ↓
审核人点击"开始审核" → in_review
       ↓
  按检查清单逐项检查
       ↓
  填写审核备注 + 评分(1-5)
       ↓
  ┌──────────┬──────────────┐
  ↓          ↓              ↓
通过       需修改         不建议交付
(approved) (needs_revision) (needs_revision + 备注)
  ↓
可选：标记已交付 (delivered)
```

## 6. 审核记录与 Prompt 迭代

每次审核应记录：
- **reviewNotes**：具体的问题描述和修改建议
- **reviewScore**：1-5 分整体质量评价
  - 5 = 可直接交付，无需修改
  - 4 = 小修后可交付
  - 3 = 基本可用但有明显不足
  - 2 = 需重新生成
  - 1 = 严重失败

### 用于 Prompt 迭代的记录规范

如果发现系统性问题（不是个例），请在 reviewNotes 中按以下格式标注：

```
[PROMPT_ISSUE] 类别：xxx
描述：AI 系统性地...
示例：在本次报告中...
建议修改：prompt 应增加/修改...
```

这些标注将作为下一版 prompt 迭代的输入。

## 7. 质量指标（供后续统计）

| 指标 | 计算方式 | 目标 |
|------|----------|------|
| 首次通过率 | approved / 总审核数 | ≥ 70% |
| 平均审核评分 | sum(reviewScore) / count | ≥ 3.5 |
| 12 章节完整率 | 结构验证通过 / 总生成数 | ≥ 90% |
| Fallback 使用率 | used_fallback=true / 总生成数 | ≤ 20% |
| 平均生成耗时 | avg(generation_time_ms) | ≤ 40s |
